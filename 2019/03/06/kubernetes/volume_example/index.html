  containers:
  - image: busybox

    volumeMounts:
    - mountPath: /producer_dir

    - -c
    - echo "hello world" > /producer_dir/hello; sleep 3000

    volumeMounts:
    - mountPath: /consumer_dir
      name: shared-volume
    args:
    - /bin/sh
    - -c
    - cat /consumer_dir/hello; sleep 30000

  volumes:
  - name: shared-volume
    emptyDir: {}

---------------------------------------hostPath Volume -----------------------------------------
hostPath Volume 的作用是将 Docker Host 文件系统中已经存在的目录 mount 给 Pod 的容器。大部分应用都不会使用 hostPath Volume，因为这实际上增加了 Pod 与节点的耦合，限制了 Pod 的使用。
如果 Pod 被销毁了，hostPath 对应的目录也还会被保留，从这点看，hostPath 的持久性比 emptyDir 强。

PersistentVolume (PV) 是外部存储系统中的一块存储空间，由管理员创建和维护。与 Volume 一样，PV 具有持久性，生命周期独立于 Pod。

PersistentVolumeClaim (PVC) 是对 PV 的申请 (Claim)。PVC 通常由普通用户创建和维护。需要为 Pod 分配存储资源时，用户可以创建一个 PVC，指明存储资源的容量大小和访问模式（比如只读）等信息，Kubernetes 会查找并提供满足条件的 PV。

-----------------------------NFS PersistentVolume----------------------------------------
[root@k8smaster1 volume]# showmount -e
Export list for k8smaster1:
/root/nfsdir 10.175.245.0/24
[root@k8smaster1 volume]#

[root@k8smaster1 volume]# cat nfs-pv1.yml
apiVersion: v1
kind: PersistentVolume
metadata:
  name: mypv1
spec:
  capacity:
    storage: 1Gi
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Recycle
  storageClassName: nfs
  nfs:
    path: /root/nfsdir/pv1
    server: 10.175.245.133
[root@k8smaster1 volume]#
[root@k8smaster1 volume]#
[root@k8smaster1 volume]# cat nfs-pvc1.yml
kind: PersistentVolumeClaim
apiVersion: v1
metadata:
  name: mypvc1
spec:
  accessModes:
    - ReadWriteOnce
  resources:
    requests:
      storage: 1Gi
  storageClassName: nfs
[root@k8smaster1 volume]#
[root@k8smaster1 volume]#
[root@k8smaster1 volume]# kubectl apply -f nfs-pv1.yml
persistentvolume/mypv1 created
[root@k8smaster1 volume]#
[root@k8smaster1 volume]# kubectl get pv
NAME    CAPACITY   ACCESS MODES   RECLAIM POLICY   STATUS      CLAIM   STORAGECLASS   REASON   AGE
mypv1   1Gi        RWO            Recycle          Available           nfs                     13s
[root@k8smaster1 volume]#
[root@k8smaster1 volume]# kubectl apply -f nfs-pvc1.yml
persistentvolumeclaim/mypvc1 created
[root@k8smaster1 volume]# kubectl get pv
NAME    CAPACITY   ACCESS MODES   RECLAIM POLICY   STATUS   CLAIM            STORAGECLASS   REASON   AGE

[root@k8smaster1 volume]#

Pod 中使用存储
[root@k8smaster1 volume]# cat pod1.yml
kind: Pod
apiVersion: v1
metadata:
  name: mypod1
spec:
  containers:
    - name: mypod1
      image: busybox
      args:
      - /bin/sh
      - -c
      - sleep 30000
      volumeMounts:
      - mountPath: "/mydata"
        name: mydata
  volumes:
    - name: mydata
      persistentVolumeClaim:
        claimName: mypvc1
[root@k8smaster1 volume]#
[root@k8smaster1 volume]# kubectl apply -f pod1.yml
pod/mypod1 created
[root@k8smaster1 volume]#

验证 PV 是否可用
[root@k8smaster1 volume]# showmount -e
Export list for k8smaster1:
/root/nfsdir 10.175.245.0/24
[root@k8smaster1 volume]# ls /root/nfsdir/pv1/
[root@k8smaster1 volume]#
[root@k8smaster1 volume]#
[root@k8smaster1 volume]# kubectl exec mypod1 touch /mydata/hello_test
[root@k8smaster1 volume]# ls /root/nfsdir/pv1/
hello_test
[root@k8smaster1 volume]#

如果不再需要使用 PV，可用删除 PVC 回收 PV.



与之对应的是动态供给（Dynamical Provision），即如果没有满足 PVC 条件的 PV，会动态创建 PV。相比静态供给，动态供给有明显的优势：不需要提前创建 PV，减少了管理员的工作量，效率高。
动态供给是通过 StorageClass 实现的，StorageClass 定义了如何创建 PV
